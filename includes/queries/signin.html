Gestion des connexions

Description rapide: Cette page permet de connecter un utilisateur. Elle se divise en 5 sections, définies par la variable $_POST['method']:

Lors de la connexion de l'utilisateur, une ligne est ajoutée dans la base de donnée, sauvegardant une clé et un token qui seront enregistré dans les cookies si l'utilisateur désire cette méthode d'authentification. A la première session de cet utilisateur, une fonction vérifie si il y a un token ou une clé dans les cookies, et s'ils existent, on sauvegarde dans les variables de session des informations telles que nom, prénom, e-mail.

  • Connexion par Facebook
  • Connexion par Github
  • Connexion par Google
  • Connexion par Twitter
  • Connexion par Let's Dev !

Type de données renvoyées: Tableau JSON de la structure suivante:

  • status (string): Statut de la requête (success: la requête s'est bien passée et l'utilisateur a été connecté; error: il y a eu une erreur lors de la requête)
  • messages (array): Liste des messages d'erreur ou de succès

Attention: N'oubliez pas de renseigner $_POST['method'], sinon, une erreur sera générée et le tableau renvoyé sera illisible.
